## 1. 문제 요약

- NxM 연구소에 빈 칸, 벽을 채워 상하좌우로 퍼지는 바이러스를 막고자 한다.
- 벽은 꼭 3개를 세워야 한다.
- 0 : 빈 칸 / 1 : 벽 / 2 : 바이러스
- 얻을 수 있는 안전 영역의 최댓값을 구하라.

## 2. 입출력

### [입력]

- N, M (3<=N,M<=8)
- 2<=바이러스의 개수<=10
- 빈 칸의 개수 3개 이상 

## 3. 알고리즘

빈 칸의 최소 개수 3개, 바이러스 최소 개수 2개이기에, 총 64의 바이러스가 퍼지기 전, 최대로 가질 수 있는 빈칸 개수는 59이다.
59에서 순서를 배제한 3개를 고르는 경우의 수 
  : 59C3 = (59*58*57) / (3*2*1) = 32,509

- 입력 받을 시 초기 빈칸 개수, 각 빈칸 위치 정보 리스트, 각 바이러스 위치 정보 리스트 초기화 
- for문 3개로 조합 계산
  - 각 빈칸 위치를 first, second, third로 구분하고, 각 빈칸에 벽을 세운다고 가정
  - 벽을 가정한 후, 각 바이러스 위치로부터 상하좌우 얼마나 뻗어나갈 수 있는지 bfs 계산
    - 📖bfs 방법 1 : 모든 바이러스의 위치를 큐에 모두 담기 + 방문처리 후 bfs를 시작
    - 📖bfs 방법 2 : 각 바이러스의 위치에 대해 각각 bfs 돌기 + visit 방문처리 공유
  - 초기 빈칸 개수 - 벽 3개 - 바이러스 퍼진 개수의 최소값이 정답!
